================================================================================
DATABASE MODELS
================================================================================

1. EventRegistration
   - full_name: CharField (max_length=200)
   - email: EmailField
   - student_id: CharField (max_length=50, optional)
   - event: ForeignKey to Event
   - created_at: DateTimeField (auto)
   Purpose: Store event registration data

2. Membership
   - name: CharField (max_length=200)
   - email: EmailField
   - phone: CharField (max_length=20)
   - role: CharField (choices: student, researcher, faculty)
   - interest: TextField (optional)
   - created_at: DateTimeField (auto)
   Purpose: Store membership applications

3. News
   - title: CharField (max_length=200)
   - content: TextField
   - image: ImageField (optional)
   - is_published: BooleanField
   - created_at: DateTimeField (auto)
   - updated_at: DateTimeField (auto)
   Purpose: Manage news articles

4. Event
   - title: CharField (max_length=200)
   - description: TextField
   - event_type: CharField (choices: workshop, course, seminar, ctf, conference, other)
   - event_date: DateTimeField (optional)
   - location: CharField (max_length=200, optional)
   - image: ImageField (optional)
   - is_published: BooleanField
   - registration_open: BooleanField
   - created_at: DateTimeField (auto)
   - updated_at: DateTimeField (auto)
   Purpose: Manage events and courses

5. GalleryImage
   - title: CharField (max_length=100)
   - caption: CharField (max_length=200, optional)
   - image: ImageField
   - is_active: BooleanField
   - order: PositiveIntegerField
   - created_at: DateTimeField (auto)
   Purpose: Manage gallery images

6. ContactMessage
   - full_name: CharField (max_length=200)
   - email: EmailField
   - phone: CharField (max_length=20, optional)
   - subject: CharField (max_length=200)
   - message: TextField
   - is_read: BooleanField
   - created_at: DateTimeField (auto)
   Purpose: Store contact form submissions

================================================================================
DEPLOYMENT NOTES
================================================================================

Static Files:
- Run: python manage.py collectstatic
- Static files collected to: staticfiles/

Media Files:
- User uploads stored in: media/
- Configure MEDIA_ROOT and MEDIA_URL in settings.py

Database:
- Development: SQLite (db.sqlite3)
- Production: Consider PostgreSQL or MySQL

Environment:
- DEBUG = True (development)
- DEBUG = False (production)
- Configure ALLOWED_HOSTS for production
